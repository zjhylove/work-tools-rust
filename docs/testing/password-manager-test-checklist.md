## 测试步骤

### 1. 导入插件
- [ ] 打开 Work Tools 应用
- [ ] 点击左侧插件商店按钮 (🧩)
- [ ] 点击"导入插件"按钮
- [ ] 选择 `plugins/password-manager/password-manager.wtplugin.zip`
- [ ] 验证插件导入成功
- [ ] 验证插件图标出现在侧边栏 (🔐)

### 2. 测试添加密码功能
- [ ] 点击密码管理器插件
- [ ] 点击"添加密码"按钮
- [ ] 填写表单:
  - 服务名称: `测试服务`
  - 用户名: `test@example.com`
  - 密码: `password123`
  - URL: `https://example.com`
- [ ] 点击"保存"按钮
- [ ] 验证:
  - [ ] 无错误提示
  - [ ] 新密码出现在列表中
  - [ ] 显示正确的服务名称和用户名
  - [ ] 控制台无错误日志

### 3. 测试更新密码功能
- [ ] 在密码列表中点击刚才添加的密码条目
- [ ] 验证详情视图显示正确的密码信息
- [ ] 点击"编辑"按钮
- [ ] 修改密码字段为: `newpassword456`
- [ ] 点击"保存"按钮
- [ ] 验证:
  - [ ] 无错误提示
  - [ ] 密码已更新
  - [ ] updated_at 时间已更新
  - [ ] 控制台显示调用 `update_password` 方法

### 4. 测试删除密码功能
- [ ] 选择一个密码条目
- [ ] 点击"删除"按钮
- [ ] 在确认对话框中点击"删除"
- [ ] 验证:
  - [ ] 密码从列表中移除
  - [ ] 控制台显示调用 `delete_password` 方法
  - [ ] 无错误提示

### 5. 测试复制密码功能
- [ ] 创建一个测试密码条目
- [ ] 点击密码条目右侧的"复制"按钮
- [ ] 验证:

### 6. 测试打开链接功能

## 修复记录
- ✅ 修复保存密码失败的问题 (2026-03-03)
  - 原因: 调用了不存在的 `save_password` 方法
  - 解决: 使用 `add_password` 和 `update_password` 方法
- ✅ 修复链接无法打开的问题 (2026-03-03)
  - 原因: `open_url` 命令未注册到 Tauri
  - 解决: 在 lib.rs 中注册 `open_url` 命令
